# FlipWord 商品化改造规划

# 编写和更新时间 2026.04.15

> 核心目标：从“开发者插件”升级为“C 端沉浸式资讯阅读产品”
> 当前优先验证：留存 + 传播
> 核心承诺：把日常资讯流，渐进读成英语环境

---

## 改造总览

| # | 改造项 | 服务的商品化标准 | 优先级 | 难度 |
|---|--------|------------------|--------|------|
| 1 | ~~首次体验：可控的 30 秒体验引导~~ | S1 + S2 | P0 | DONE |
| 2 | ~~Popup 首页重构：阅读状态面板~~ | S1 + S3 | P0 | DONE |
| 3 | ~~页面存在感与控制：阅读空气层~~ | S2 | P0 | DONE |
| 4 | ~~成长反馈体系重写：从系统统计到阅读进步~~ | S3 | P1 | DONE |
| 5 | ~~品牌文案与语气统一~~ | 全局 | P1 | DONE |
| 6 | ~~可分享周报卡片~~ | S4 | P2 | DONE |

## Spec 1: 首次体验 — 可控的 30 秒体验引导

### 现状问题
- 首次进入时，用户先面对词库、等级、主题等设置项，先有“配置感”，没有“产品感”
- 如果直接静默改动当前页面，会像插件偷偷动了网页，损伤信任感
- 现有引导完成条件过弱，不适合承接真正的 first wow moment

### 目标
安装后 30 秒内，用户在真实资讯页面上感受到“页面开始长出英语”，同时明确知道自己随时可以暂停、恢复原文或跳过当前网站。

### 方案

**原则**
- 不做静默激活
- 首次体验必须和控制权同时出现
- 不要求用户先理解词库、等级、主题这些概念

**阶段一：首个可体验页面出现轻提示卡**
- 当用户进入首个符合条件的资讯页时，右下角出现极简 first-run 卡片
- 卡片文案示例：
  ```
  试试让这页慢慢长出英语
  [立即体验] [低强度] [跳过本站]
  ```
- 不遮挡正文，8-10 秒不操作则收成一个小点，hover 可再次展开

**阶段二：进入体验预设**
- 点击“立即体验”后，启用推荐预设：
  - 沉浸强度：L1
  - 默认预设：资讯模式（优先 `news + tech`，后续可扩展 `business`）
- 页面只替换 5-10 个词，强调“轻度、可读、不会毁页面”
- 同时将卡片切换为控制态：
  ```
  已进入体验
  [恢复原文] [暂停] [继续]
  ```

**阶段三：首次点击词后的轻提示**
- 用户第一次点击替换词时，decode panel 正常弹出
- panel 底部仅增加一行一次性提示：
  `左右切换 · esc 关闭 · 这是你解码的第一个词`

**阶段四：引导完成条件**
- 不再使用“关闭一次 decode panel 就算完成”
- `onboarded` 只在以下任一显式行为发生时写入：
  - 用户点击过任意替换词
  - 用户在 first-run 卡片中点击“继续”
- 如果用户只是忽略或关闭提示，不标记完成；后续最多再出现 2 次

### 要删掉或后移的
- 不做“先选词库”的 onboarding
- 不做“先选难度”的 onboarding
- 不做预测试
- 不做长动画 walkthrough

### 涉及文件
- `src/onboarding/index.ts` — 重写为 first-run 提示卡，而不是设置向导
- `src/content.ts` — 首次体验预设、显式完成逻辑
- `src/core/constants.ts` — 默认首用预设
- `src/decode/index.ts` — 一次性轻提示

### 反商品化检查
- [ ] 首次体验前，页面不会被静默修改
- [ ] 用户在看到页面变化的同时，也能看到暂停/恢复入口
- [ ] 首次体验默认像“资讯阅读模式”，不是考试词库模式
- [ ] 用户不需要先做任何配置选择

## Spec 2: Popup 首页重构 — 从控制台到阅读状态面板

### 现状问题
- 首页同时塞入统计、等级、主题、词库，像设置中心，不像产品首页
- 用户打开 popup 后，无法在 3 秒内回答“它现在在帮我什么”
- 当前数据结构更偏系统后台，不够贴近“阅读状态”

### 目标
打开 popup 后，用户 3 秒内能看懂三件事：
1. 现在是否开启
2. 当前在什么阅读模式里
3. 今天有没有比昨天更进入英语环境

### 新首页布局

```text
┌──────────────────────────────┐
│ FlipWord              [ON]   │
├──────────────────────────────┤
│ 今日沉浸                      │
│ 12 个表达 · 科技资讯         │
│ 比昨天多 3 个                │
├──────────────────────────────┤
│ 本周趋势             第 5 天  │
│ ▁▂▃▅▆█▇                     │
├──────────────────────────────┤
│ 当前模式                      │
│ 轻度 · 资讯模式 · editorial   │
│ [调整设置 →]                 │
└──────────────────────────────┘
```

### 关键改造

**核心状态优先**
- 首页只保留：
  - 今日吸收表达数
  - 今日活跃阅读领域
  - 对比昨日变化
  - 7 天趋势
  - 当前模式摘要

**设置后移**
- 等级、主题、词库从首页移入“调整设置”折叠区
- 一级 tab 精简为：
  - `主页`
  - `统计`
  - `设置`

**领域信息来源改造**
- “科技/商务/学术”等领域，不再从 active word banks 直接推断
- 改为从最近 24h 实际遇到的词和其 bank/domain 标签聚合得出
- 目标是让首页说的是“你今天读了什么”，不是“你配置了什么”

**新的首页消息**
- 新增 `GET_TODAY_SUMMARY`：
  - 今日吸收表达数
  - 昨日对比
  - 今日活跃领域
  - 当前模式摘要

### 涉及文件
- `src/popup/index.tsx` — 首页布局重构
- `src/popup/stats.tsx` — 承接更多详细数据
- `src/popup/more.tsx` 或新 `src/popup/settings.tsx` — 设置承接
- `src/background.ts` — 新增 `GET_TODAY_SUMMARY`
- `src/core/i18n.ts` — 首页文案更新

### 反商品化检查
- [ ] 首屏不需要滚动就能理解核心状态
- [ ] 首页像“阅读状态”，不像“插件设置”
- [ ] 领域信息来自真实使用，而不是静态配置
- [ ] 设置存在，但不抢视觉重心

## Spec 3: 页面存在感与控制 — 阅读空气层

### 现状问题
- 暂停、跳过等操作必须打开 popup，控制权离正文太远
- 页面上的计数器、combo badge、全屏特效更像游戏 HUD 或调试信息
- 在严肃资讯阅读场景下，这些元素会拉低质感

### 目标
页面上的 FlipWord 只表现为两件事：
1. 被替换的词本身
2. 一个低调但清晰的页面级控制入口

### 方案

**删除**
- 左上角 `Flip -> XX` 计数器
- 右上角持久 combo badge
- 全屏 canvas 庆祝效果

**保留但降调**
- first-blood 的轻微 pulse 保留
- 分数 float 保留，但字号、透明度和持续时间都降低
- combo 事件只保留短暂轻反馈，不再形成 HUD

**新增：右下角极简状态指示器**
- 默认态：8x8 半透明圆点，慢速呼吸
- hover 态：展开为 pill，显示：
  - `flipword`
  - 当前页已替换词数
  - `[暂停]`
  - `[恢复原文]`
  - `[跳过本站]`
- 暂停后切换为灰色状态，可直接恢复
- `prefers-reduced-motion` 下改为静态

**首用体验联动**
- 首次体验卡和页面级状态指示器使用同一套存在感设计
- 首用结束后，first-run 卡片退出，保留极简状态指示器

### 涉及文件
- `src/crit/css-effects.ts` — 删除 HUD 型效果，保留轻反馈
- `src/content.ts` — 删除旧调用，接入页面级控制
- 新建 `src/presence/indicator.ts` — 页面状态指示器
- `src/fx/index.ts` — 移除全屏 canvas 庆祝

### 反商品化检查
- [ ] 页面截图看起来像正常网页，而不是插件仪表盘
- [ ] 用户不打开 popup 也能暂停、恢复、跳过本站
- [ ] 页面存在感低调，但控制入口始终可达
- [ ] 游戏化反馈不会压过阅读体验

## Spec 4: 成长反馈体系重写 — 从系统统计到阅读进步

### 现状问题
- 当前统计更像机器后台，不像用户能感知的成长
- 用户看不到“这周我比上周更能吃下哪些资讯表达”
- “全部词汇”列表信息密度高，但成就感弱

### 目标
Stats 页要让用户感受到：
“我在真实资讯阅读中，正在逐渐适应英语表达。”

### 新 Stats 页结构

```text
┌──────────────────────────────┐
│ 本周摘要                      │
│ 吸收 47 个表达 (+12 vs 上周)  │
│ 主要领域：科技、商务          │
│ 掌握率：32% -> 38%           │
├──────────────────────────────┤
│ 7 天趋势                      │
│ ▁▂▃▅▆█▇                     │
├──────────────────────────────┤
│ 掌握进度                      │
│ ██████░░░░ 38% 已掌握        │
├──────────────────────────────┤
│ 最近掌握                      │
│ environment · infrastructure │
│ deploy · performance         │
└──────────────────────────────┘
```

### 关键改造

**周对比**
- 新增“本周 vs 上周”摘要
- 用文字直接表达变化，不要求用户自己心算

**领域标签来源改造**
- 不再从 active word banks 推断主要领域
- 需要在曝光记录时，轻量聚合词条来源领域或 bank 标签
- 只存聚合结果，不存原文章内容

**掌握率**
- 使用：
  - `mastered / (seen + learning + reviewing + mastered)`
- `new` 不计入分母

**“最近掌握”替换“全部词汇”**
- 默认展示最近 mastered 的 10-20 个词
- 完整列表放二级展开

**文案升级**
- 所有 stats 文案从“后台统计”转成“阅读进步”
- 目标是让用户读完一屏就知道：
  - 我在哪些语境里更熟了
  - 我这周确实比上周强了一点

### 涉及文件
- `src/popup/stats.tsx` — 布局和文案重写
- `src/background.ts` — 新增本周/上周摘要接口
- `src/core/storage.ts` — 新增领域聚合历史
- `src/core/i18n.ts` — 新文案

### 反商品化检查
- [ ] 首屏能看到“这周比上周更强了”
- [ ] 领域标签来自真实阅读，而不是用户配置
- [ ] “最近掌握”比“全部词汇”更有成就感
- [ ] 数据表达的是阅读进步，不是系统内部状态

## Spec 5: 品牌文案与语气统一

### 现状问题
- 游戏化术语、系统术语、品牌语气混用
- 中英切换不彻底，部分页面像半成品
- 部分用词偏工具化、开发者化

### 目标
所有用户可见文案都像同一个品牌写出来的：
冷感、克制、清楚、不讨好、不解释过度。

### 关键改动

| 现在 | 改为 | 原因 |
|------|------|------|
| 入侵等级 | 沉浸强度 | 更适合阅读产品 |
| Invasion Level | Immersion Level | 同上 |
| 自定义 | 我的词汇 | 降低工具味 |
| 更多 | 设置 | 更直接 |
| TODAY/TOTAL/CLEARED/BEST COMBO | 移出首页 | 不再像后台面板 |
| v0.1.0 | 移除 | C 端首页不需要版本号 |
| Infiltrate/Spread/Invade/Override | 移除 | RPG 感太强 |

### 语气原则
- 中文：简洁，不卖萌，不喊口号
- 英文：克制，避免全大写喊叫感
- 错误提示：说明原因 + 给出下一步
- 成功提示：确认结果，不做情绪表演

### 重点清理范围
- popup 首页
- stats 页
- decode panel
- 更多/设置页
- 首次体验提示

### 涉及文件
- `src/core/i18n.ts`
- `src/core/constants.ts`
- `src/popup/index.tsx`
- `src/popup/stats.tsx`
- `src/popup/more.tsx`
- `src/decode/index.ts`

### 反商品化检查
- [ ] 所有文案读起来像同一个品牌
- [ ] 没有残留的开发者术语或 RPG 术语
- [ ] 中英切换后体验完整，不会“一半是中文一半是英文”

## Spec 6: 可分享周报卡片

### 现状问题
- 当前没有一个可以直接截图或分享的产出物
- 用户很难用一句话向别人说明 FlipWord 在帮他什么

### 目标
提供一张低门槛、可截图、可转发的周报卡片，让用户能自然表达：
“我最近刷资讯时，页面真的在慢慢长出英语。”

### 方案

**卡片内容**
```text
FlipWord Weekly
2026.04.07 - 2026.04.13

这周你在资讯阅读中
吸收了 47 个英文表达

主要领域：科技 · 商务
掌握率：32% -> 38%
连续学习：5 天
```

**设计原则**
- 以“感受”和“变化”为主，不堆原始数字
- 一张图里只保留最能支持传播的话
- 不强塞网址；若没有正式站点，只保留品牌名

**触发方式**
- v1 只做手动生成：
  - Stats 页底部 `[生成本周卡片]`
- 自动周日提醒后移，先验证用户是否真的会分享

**生成方式**
- 使用纯 canvas 绘制，避免额外依赖
- 输出 PNG，可保存或复制

### 涉及文件
- 新建 `src/popup/weekly-report.tsx`
- `src/popup/stats.tsx` — 增加入口
- `src/background.ts` — 聚合周数据

### 反商品化检查
- [ ] 一张图能讲清 FlipWord 的价值
- [ ] 内容以变化和感受为主
- [ ] 分享素材看起来像品牌产物，不像后台截图
- [ ] v1 不为了“自动化”而增加复杂度

## 执行顺序建议

```text
Phase 1 — 先建立“值不值 + 可控”的第一印象（1-2 天）
  ├── Spec 1: 首次体验
  ├── Spec 3: 页面存在感与控制
  └── Spec 5: 品牌文案与语气统一

Phase 2 — 再建立“我在变强”的持续感（2-3 天）
  ├── Spec 2: Popup 首页重构
  └── Spec 4: 成长反馈体系重写

Phase 3 — 最后做“我愿意发给别人”的传播物（1-2 天）
  └── Spec 6: 可分享周报卡片
```

## 不做清单（YAGNI）

以下内容当前不做：
- 静默激活并直接改动当前页面
- 首次体验预测试
- 动画 walkthrough
- 社交排行榜
- 每日推送通知
- 自动周报提醒
- 多语言扩展（日/韩/法等）
- 为传播先上复杂分享系统
- LLM 消歧义商品化包装

---

## 一句话约束

FlipWord 不是“功能很多的英语插件”，而是一个把资讯流渐进读成英语环境的阅读产品。
